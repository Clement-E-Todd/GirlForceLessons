    Debugging Our Programs

    As our programs become more and more complex as we introduce more variables and conditional behaviour, it becomes increasingly difficult to accurately understand what our program will do simply by reading the code. Very often, need to pause the program and let it run line by line so that we can see exactly what it's doing and why.

    This is where our IDE's debugging tools come in handy. Most IDEs allow you to set something called a "breakpoint" on any given statement in your program. When you run your program after setting a breakpoint on a statement in your code, your program will freeze until you allow it to continue. You decide at this point whether to allow the program to continue as normal or whether to step through it one statement at a time.

    Your debugger will have a number of buttons that allow you to control the flow of your program while it is paused. These buttons will include the options to "continue", "step over", "step into" and "step out". The exact names of these features may vary from one IDE to another, and most IDEs will also include other features not listed here (though these four are the most important ones). "Continue" will allow you to unpause the program and continue running it as normal, while "step over" runs the current line of code only, steps to the next statement and pauses again. This is an extremely useful feature for understanding the flow of your program as the computer sees it.

    More Variable Types

    In the previous lesson we learned how to create string-type variables, which store text data. However, there is a fairly strict limit to how much we can do with only strings. For example, it is not possible to perform mathematical operations with string values; if we have a string variable with a value "256", the computer understands that to be a series of characters: '2', '5' and '6'. It does not see it as the numerical value of two hundred and fifty six. It it important to develop a knowledge of a variety of variable types and an understanding of what types to use in a given situation.

    Numerical Variable Types

    Mathematical operations are very commonly needed when writing a program, so our numerical variable types will be used very frequently. The three numerical types that we will dicuss for now are int, float and double.

    An int (or integer) variable can store whole numbers, both positive an negative. int variables are very handy when keeping track of information like a player's score or the number of treasures that they have found, as these numbers would typically never require partial values (for example, the player can't find half of a treasure). We can declare a new int variable the same way that we declare a string variable, but we replace the word 'string' with 'int':
    int score = 0;

    When we need to store decimal information, we can use either a float-type variable or a double-type variable. The main difference between these two variable types is that a double takes twice as much memory as a float. This allows a double to be significantly more precise that a float when it comes to decimal numbers with a large number of digits after the period, but because it takes more of our computer's resources it should only be used when that precision is needed. We can declare a new double like so:
    double pi = 3.1415;
    However, this will not work for float values. We will get an error if we attempt to assign a float value like this:
    float pi = 3.1415;
    This is because when we write a decimal number in C#, the computer will assume that it is a double value; it errs on the side of precision rather than memory efficiency. To fix this, all we have to do is add an 'f' to the end of the decimal number to let the comuter know that the number should be treated as a float:
    float pi = 3.1415f;


    Other Variable Types

    Apart from string and numerical variable types there are a couple of basic variable types that are commonly used: char and bool.

    While a string contains a series of characters to form a line of text, a char is its simpler counterpart, containing only one single character. That character can be any unicode value, including letters, numbers and symbols. As you may recall, we put double quotes around our text when assigning a string value. When assigning a char value, we use single quotes instead:
    char symbol = 'â™ª';

    Finally, a bool (or 'boolean') is a variable that can contain only one of two values: true or false. This is very useful in a number of cases, for instance when you have a feature that can be turned on and off:
    bool isMusicEnabled = true;

    You can also assign a true or false value to the boolean by using a condition (like we saw when writing 'if' statements). For example, if we want to keep track of whether or not the player's name is Tina using the condition name == "Tina", but we don't want to use the processing power to calculate that condition more than once, we can store it in a variable like so:
    bool isPlayerTina = name == "Tina";
    TIP: If that line of code is visually confusing to you, you can also wrap the condition in parenthesis like you would in an 'if' statement to make it visually separate, but this is optional.
    bool isPlayerTina = (name == "Tina");

    Once you have stored a true or false value in a bool, you can use it in place of a condition in an 'if' statement or anywhere else that a condition is used: 

    if (isPlayerTina)

}
